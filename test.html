I'm Brian..

I venture up the stack, down the stack... towards easier stacks.

I work as an SRE to keep developer and user-facing platforms secure, stable, and graceful when they degrade. I've previously worked as a platform engineer, in a dev(sec)ops team, and as a backend developer.

Beyond keeping stable, I look for ways that software can make the lives of users easier. We all have too much going on, and cognitive switches are time sinks. My goal is to create software that fades into the background so that you can do what you love or produces value. Delight and utility over engagement.

I dabble a lot. Many of my repos are PoC type things, intended to work just enough to get a feel for the toolchain, possibilities, and applicability. I try to keep things MIT/BSD licensed so you can take things and run. Contact me if you'd like to explore more.

When not immersed in code, I spend time in my garden, with my cats, and exploring art.

Cloud-based dev environments

Low-code firewall rules

Firewall rule languages are incredibly high-context. Since I only make changes 2x per year, I build a tool in blockly so I can visually build my firewall rules. The tool translates blocks into the text rules my devices understand.

E-paper display

ESP-32 code to display content to a 3-color e-ink display. Stationed outside of my door to let my wife know I'm in a meeting. Memory constraints prevent decoding even the slimmest bit-depth pngs.

ebpf/uprobe tracing

An extremely minimal setup with go and clang to use the linux kernel features to monitor function calls in a general program without tools like gdb. This project watches executions and tallies counts of argument values.

Policy based (timed) file deletion

S3 provides a great way to apply a policy that deletes files after some expiry time. It's unfortunate that local filesystems haven't caught up, but the linux inotify API (and the fsnotify package in golang) take up some slack. This project seeks to remove file after some amount of time via OPA policies. It works, but doesn't scale. The memory footprint to hold the paths quickly blows up as you churn 10k+ of files.
